EXECUTIVE SUMMARY
─────────────────
The KitKat Promo Admin System is an internal web-based platform built to
manage the KitKat promotional campaign (Official Partner of Formula 1®).

It allows staff to register participants, manage a points-based rewards
program, handle prize redemptions, and monitor all activity. Participants
accumulate points and can exchange them for prizes through staff-assisted
redemptions. A public leaderboard lets participants check their own ranking
using a unique ID they receive upon registration.

The system is fully bilingual-ready, role-protected, and includes a
live countdown screen to the promotion event date (March 4, 2026).

2.1  LOGIN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Who can use it: All users (entry point to the system)

What it does:
  - Staff log in with a username and password.
  - The session lasts 8 hours. After 8 hours, the user is automatically
    logged out and must log in again.
  - Sessions are stored securely in the browser — no visible token is exposed.
  - After 5 failed login attempts in 15 minutes, the user is temporarily
    blocked to prevent brute-force attacks.
  - Logging out immediately invalidates the session, even if the 8-hour
    window has not expired.

Important notes for business:
  - Usernames and passwords are auto-generated by the system when a new
    staff account is created (see Section 2.7).
  - There is no self-service password reset for staff — only a Super Admin
    can reset a password.

2.3  REGISTER PARTICIPANT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Who can use it: Staff & Super Admin

Purpose:
  Enrol a new person into the KitKat promotion. Identity is verified via
  OTP (one-time password) codes sent to both the participant's phone and
  email before the account is created.

How it works — step by step:

  STEP 1 — Enter participant information
    Staff fills in the following fields:
      • First Name
      • Last Name
      • Government ID  (exactly 11 digits, numbers only)
      • Phone Number   (9–15 digits, no spaces or symbols, e.g. 995555123456)
      • Email Address

    Before proceeding, the system automatically checks:
      - Is this phone number already registered? → Error if yes.
      - Is this email already registered? → Error if yes.
      - Is this Government ID already registered? → Error if yes.
    This prevents duplicate registrations.

  STEP 2 — Phone Verification
    A 6-digit code is sent via SMS to the participant's phone.
    Staff asks the participant for the code and enters it into the system.
    The code expires after a configurable time. A "Resend" button is
    available, but has a 60-second cooldown to prevent abuse.

  STEP 3 — Email Verification
    After phone is verified, a 6-digit code is sent to the participant's
    email address. Staff enters it to confirm the email.
    The same resend cooldown applies.

  STEP 4 — Registration Complete
    Once both codes are verified, the participant account is created.
    The system generates a Unique ID (format: KK-XXXXXX, e.g. KK-AB1234).
    This Unique ID is immediately sent to the participant:
      • Via SMS to their phone number
      • Via email to their email address

    The success screen shows:
      • Participant's full name
      • Their Unique ID
      • Confirmation that the ID was sent

    A "Register Another Participant" button is available to immediately
    start the next registration.

What is a Unique ID?
  A Unique ID is a permanent identifier for the participant in this promo.
  It is used to:
    - Look up a participant in the system (by staff)
    - Redeem prizes
    - Check their position on the public leaderboard (by the participant)
  The format is always: KK- followed by 6 alphanumeric characters.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Who can use it: Staff & Super Admin (lock/unlock is Super Admin only)

Purpose:
  Central screen for looking up participants and performing actions
  on their account (add points, lock/unlock).

Layout:
  The screen is split into two panels:
    LEFT  — Search results / participant list
    RIGHT — Details and action buttons for the selected participant

Functionality:

  SEARCH
    Staff can search by:
      • Unique ID (e.g. KK-AB1234)
      • Phone number
      • Email address
    Partial matches are supported. Leaving the search box empty and clicking
    Search loads the full list of all participants (most recent first).

  PARTICIPANT DETAILS (shown when a participant is selected)
    • Unique ID (large, prominently displayed)
    • Full name
    • Total Points
    • Active Points
    • Status (ACTIVE or LOCKED, colour-coded)
    • Phone number
    • Email address

  ADD POINTS
    Available to: Staff & Super Admin
    Staff clicks "Add Points", enters:
      • Number of points (whole number, minimum 1)
      • Optional note (e.g. "Purchased 2 KitKat bars")
    On confirmation:
      - Both Total Points and Active Points increase by the entered amount.
      - The action is recorded in the activity log with the staff member's
        name, the participant, the points added, and the note.

  LOCK PARTICIPANT
    Available to: Super Admin only
    Purpose: Prevent a participant from redeeming prizes (e.g. suspected fraud).
    Staff must enter a reason before locking. The reason is recorded in the
    activity log. A locked participant's points balance is preserved; they
    simply cannot redeem while locked.

  UNLOCK PARTICIPANT
    Available to: Super Admin only
    Restores the participant to ACTIVE status. The unlock action is also
    recorded in the activity log.

2.5  PRIZES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Who can use it:
  - Prize catalog management: Super Admin only
  - Prize redemption: Staff & Super Admin

Purpose:
  Manage the prize catalog and process prize redemptions for participants.

THE PRIZE CATALOG
  Each prize has the following attributes:
    • Name               (required)
    • Description        (optional, up to 1,000 characters)
    • Image URL          (optional, links to an external image)
    • Cost in Points     (required, minimum 1 point)
    • Stock Quantity     (optional — leave blank for unlimited stock)
    • Status             (ACTIVE or INACTIVE)

  Prizes are displayed as cards in a grid, showing the image (if set),
  name, description, point cost, stock remaining, and status badge.

  Super Admin actions per prize:
    • Edit      — update any prize detail at any time
    • Delete    — permanently remove a prize (with confirmation dialog)
    • Activate/Deactivate — controlled via the status field when editing

  Stock management:
    If a prize has a stock quantity set, it decrements by 1 each time
    the prize is redeemed. When stock reaches 0, the Redeem button is
    automatically disabled for that prize. Setting stock to blank means
    the prize is unlimited.

PRIZE REDEMPTION
  Available to: Staff & Super Admin

  How to redeem a prize for a participant:
    1. Click "Redeem Prize" button at the top of the Prizes page.
    2. Enter the participant's Unique ID (format: KK-XXXXXX).
    3. Click Search. The system displays the participant's name and
       their current Active Points balance.
    4. The system highlights which prizes the participant can afford
       (enough active points, prize is ACTIVE, stock available).
       Prizes they cannot afford are greyed out.
    5. Click "Redeem" on the desired prize. A confirmation dialog shows
       the prize name and participant name.
    6. On confirmation:
         - Active Points are deducted by the prize's point cost.
         - Stock quantity decrements by 1 (if applicable).
         - The redemption is recorded in the activity log.
         - Total Points are NOT affected (ranking is preserved).

  Validation rules (all must pass for redemption to succeed):
    • Participant must be ACTIVE (not LOCKED)
    • Prize must be ACTIVE
    • Participant must have enough Active Points
    • Stock must be available (> 0 if limited)

THE STAFF TABLE
  Displays all staff members with the following columns:
    • Name
    • Email
    • Username
    • Role     (Super Admin / Staff)
    • Status   (ACTIVE / DISABLED)
    • Actions  (Reset Password, Deactivate/Activate)

CREATING A NEW STAFF ACCOUNT (2-step process)

  STEP 1 — Enter staff information
    Super Admin fills in:
      • First Name
      • Last Name
      • Email (must be unique — the system checks for duplicates)
      • Role (Staff or Super Admin)
    On submission, a 6-digit verification code is emailed to the address
    entered. This confirms the email is valid and accessible.

  STEP 2 — Verify email and complete account creation
    Super Admin enters the code received on the staff member's email.
    On successful verification:
      - The system auto-generates a unique username
        (derived from the staff member's name)
      - The system generates a strong random password
      - An email is sent to the staff member with their username and
        password and instructions to keep it secure
    The new account appears in the staff table immediately.

RESET PASSWORD
  Super Admin can reset any staff member's password at any time.
  A new random password is generated and emailed to the staff member.
  This action is recorded in the activity log.
  Note: Reset is disabled if the account is DISABLED.

ACTIVATE / DEACTIVATE
  Super Admin can disable or re-enable any account except their own.
  When deactivated:
    - The staff member's current session is immediately invalidated
      (they are logged out instantly, even mid-session)
    - They cannot log in until reactivated
  The action is confirmed with a dialog showing the staff member's name.

SECTION 3 — POINTS SYSTEM EXPLAINED
────────────────────────────────────────────────────────────────────────────────

Every participant has two separate point counters:

  TOTAL POINTS
    • Increases every time points are added by staff.
    • Never decreases (prize redemptions do NOT affect this counter).
    • Used exclusively for leaderboard ranking.
    • Ensures that participants who redeem prizes are not penalised in
      the competition rankings.

  ACTIVE POINTS
    • The participant's "spending balance."
    • Increases when points are added.
    • Decreases when prizes are redeemed.
    • Shown on the participant's profile and during redemption.
    • If a participant has fewer Active Points than a prize costs,
      they cannot redeem that prize.

Example:
  A participant earns 50 points.
    → Total Points: 50, Active Points: 50
  They redeem a prize costing 20 points.
    → Total Points: 50, Active Points: 30
  Their leaderboard rank is still based on 50 (Total Points).

SECTION 6 — SECURITY HIGHLIGHTS
────────────────────────────────────────────────────────────────────────────────

The following security measures are in place:

  • All pages require login — no content is accessible without authentication
  • Role enforcement — staff cannot access Super Admin features, and the
    system returns an error (not just hides buttons) if rules are bypassed
  • Rate limiting — login (5 attempts/15 min), OTP send (1/min per person),
    OTP verify (10 attempts/15 min)
  • Passwords are stored using bcrypt (one-way hashing)
  • OTP codes are hashed before being stored — even if the database is
    compromised, codes cannot be recovered
  • Sessions can be instantly revoked (logout or account disable)
  • All inputs are validated server-side — client data is never trusted
  • Sensitive data is never returned in public API responses
  • Complete audit log — every action is traceable to a specific staff member

────────────────────────────────────────────────────────────────────────────────

  Admin Panel     Web application, works in any modern browser, fully
                  responsive (works on desktop, tablet, and mobile)

  Backend         Secure API server handling all business logic and
                  data processing

  Database        PostgreSQL — industry-standard relational database
                  storing all participant, prize, staff, and log data

  Cache           Redis — used for session management and OTP storage,
                  enabling instant session revocation

  SMS             Twilio — sends OTP codes and Unique IDs to participants
                  via text message

  Email           SMTP — sends OTP codes, Unique IDs, and staff credentials
                  via email

  Hosting         Admin panel hosted on Vercel (global CDN)
                  Backend hosted on a separate server
